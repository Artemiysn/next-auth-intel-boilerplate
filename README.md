Особенности:

1. URL-based: Язык определяется по URL (например, /en/home или /ru/home) 
2. NExt.js 15 (для лучшей совместимости)
3. Язык сайта добавляется в куки пользователя (может быть потребуется какое-то соглашение вывести, согласен ли он с куками)


Как улучшить:

1. можно форматировать tailwind классы в className, например, классы которые отвечают за позиционирование писать на отдельной строчке, 
2. Можно локализовать страницы 404 и 500 ошибок
