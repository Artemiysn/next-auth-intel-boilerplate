Особенности:

1. URL-based: Язык определяется по URL (например, /en/home или /ru/home)
2. NExt.js 15 
3. Язык сайта добавляется в куки пользователя. 
4. Для авторизации используем JWT. Они безсессионы, быстры и содержат всю необходимую нам информацию.
Помещаем их в куки с параметрами для защиты от XSS:
    httpOnly: true, 
    sameSite: 'strict'

Как улучшить:

1. Можно форматировать tailwind классы в className, например, классы которые отвечают за позиционирование писать на отдельной строчке, 
2. Локализовать страницы 404 и 500 ошибок  
3. Скорее всего нужно будет создать свой UI Kit или использовать готовый, например, shadcn. Можно его создать в папке ./components/ui или просто .ui/
4. Разделить один токен безопасности, на два: access token и refresh token и скорее всего придется добавить аутентификацию через OAuth 2 (используя стороннего провайдера, например гугл). Для улучшения безопасности. Возможно стоит использовать Clerc или подобный сервис. Или библиотеку NextAuth.
5. Добавить .env файл в gitignore! (Здесь он передается для быстроты развертывания)
6. Убрать недоступные для зарегистрированных пользователей ссылки из хедера
