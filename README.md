Особенности:

1. URL-based: Язык определяется по URL (например, /en/home или /ru/home). Важно для СЕО.
2. NExt.js 15+ и немного модифицирован nextConfig (исключительно дял того, чтобы сборка прошла без проблем)
3. Язык сайта добавляется в куки пользователя. Если пользователь хотя бы один раз переключил язык, сайт это запомнит.
4. Для авторизации используем JWT. Они безсессионы, быстры и содержат всю необходимую нам информацию.
Помещаем их в куки с параметрами для защиты от XSS:
    httpOnly: true, 
    sameSite: 'strict'

Как улучшить:

1. Можно форматировать tailwind классы в className для читаемости, например, классы которые отвечают за позиционирование писать на отдельной строчке.
2. Локализовать страницы 404 и 500 ошибок  
3. Скорее всего нужно будет создать свой UI Kit или использовать готовый, например, shadcn. Можно его создать в папке @/components/ui или просто @/ui/
4. Можно разделить один токен безопасности, на два: access token и refresh token (считается, что это улучшит безопасность) и скорее всего придется добавить аутентификацию через OAuth 2 (используя стороннего провайдера, например гугл). 
6. Убрать недоступные для неавторизованных пользователей ссылки из хедера (хорошая практика)
7. Большой файл перевода можно разбить на несколько файлов для удобства
